Why All FastAPI — No Node.js

Node.js handled business logic (auth, prompts, social). FastAPI handled AI. This blueprint collapses both into one FastAPI application using APIRouter modules — same separation, zero polyglot overhead. Python's ecosystem handles everything: JWT auth, S3 uploads, PostgreSQL (asyncpg), Redis, and all ML models.

dependencies/auth.py — JWT guard

# dependencies/auth.py from fastapi.security import OAuth2PasswordBearer from jose import jwt, JWTError oauth2_scheme = OAuth2PasswordBearer(tokenUrl="/auth/login") async def get_current_user( token: str = Depends(oauth2_scheme), redis = Depends(get_redis), db = Depends(get_db) ): # Check Redis blacklist first (logout support) if await redis.get(f"blacklist:{token}"): raise HTTPException(401, "Token revoked") try: payload = jwt.decode(token, SECRET_KEY, algorithms=["HS256"]) user_id = payload.get("sub") except JWTError: raise HTTPException(401, "Invalid token") return await user_service.get(db, user_id) require_auth = Depends(get_current_user)

dependencies/rate_limit.py

async def rate_limit(request: Request, redis = Depends(get_redis)): key = f"rl:{request.client.host}" count = await redis.incr(key) if count == 1: await redis.expire(key, 60) if count > 100: raise HTTPException(429, "Rate limit exceeded") # Use as dependency on any endpoint: @router.post("/", dependencies=[Depends(rate_limit)])

Upload Flow

1
React sends multipart/form-data
POST /images/upload with UploadFile + prompt_id. FastAPI's python-multipart handles the stream.
2
FastAPI validates file
Checks content_type ∈ ALLOWED, size ≤ 10 MB, require_auth dependency confirms JWT.
3
boto3 uploads to S3
Original → originals/{uuid}.png. Pillow thumbnail → thumbnails/{uuid}.png. Both uploaded async via asyncio.to_thread.
4
S3 URLs saved to PostgreSQL
images table row inserted with s3_url, thumbnail_url, qdrant_id=None (filled later).
5
BackgroundTask triggers AI pipeline
FastAPI's BackgroundTasks calls classify_and_embed(image.id, s3_url) — no separate service call needed.
6
CLIP embedding stored in Qdrant
embed router generates 512D vector, upserts to Qdrant collection, updates images.qdrant_id in PG.

s3_client.py

# s3_client.py import boto3, asyncio from config import settings s3 = boto3.client("s3", region_name=settings.AWS_REGION, aws_access_key_id=settings.AWS_ACCESS_KEY_ID, aws_secret_access_key=settings.AWS_SECRET_ACCESS_KEY ) async def upload_to_s3(data: bytes, key: str, content_type: str) -> str: await asyncio.to_thread( s3.put_object, Bucket=settings.S3_BUCKET_NAME, Key=key, Body=data, ContentType=content_type, ) return f"https://{settings.S3_BUCKET_NAME}.s3.{settings.AWS_REGION}.amazonaws.com/{key}" async def delete_from_s3(key: str): await asyncio.to_thread(s3.delete_object, Bucket=settings.S3_BUCKET_NAME, Key=key)

routers/ai/embed.py — CLIP Embedding

import open_clip, torch from PIL import Image import io, httpx router = APIRouter(tags=["ai"]) # Load once at startup (lifespan event in main.py) model, _, preprocess = open_clip.create_model_and_transforms( "ViT-B-32", pretrained="openai" ) model.eval() @router.post("/embed") async def embed_image(image_url: str, image_id: UUID, db = Depends(get_db)): # Fetch image from S3 URL async with httpx.AsyncClient() as c: r = await c.get(image_url) img = preprocess(Image.open(io.BytesIO(r.content))).unsqueeze(0) with torch.no_grad(): embedding = model.encode_image(img).squeeze().tolist() # Upsert to Qdrant qdrant_id = str(uuid4()) await qdrant.upsert("prompt_images", points=[ PointStruct(id=qdrant_id, vector=embedding, payload={"image_id": str(image_id)}) ]) # Save qdrant_id back to PG await image_service.set_qdrant_id(db, image_id, qdrant_id) return {"qdrant_id": qdrant_id}

⚠ One process, not microservices
By going all-FastAPI you eliminated the Node.js↔FastAPI HTTP calls entirely. The AI functions are just Python imports inside the same process. Splitting into microservices later is still possible (each router becomes its own FastAPI app) — but don't do it now.
